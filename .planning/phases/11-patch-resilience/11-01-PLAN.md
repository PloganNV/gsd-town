# Phase 11 Plan 01: Patch Resilience — Doctor Command

## Objective

Add a `gsd-town doctor` subcommand that checks and optionally repairs the two GSD integration points that npm postinstall manages: the execute-phase.md dispatch patch and the ~/.claude/skills/gsd-town-setup/ skill directory.

## Scope

1. `gsd-town doctor` — checks patch presence, reports PASS/FAIL per check, exits 0 (healthy) or 1 (unhealthy)
2. `gsd-town doctor --fix` — re-runs postinstall logic to repair missing patches
3. `gsd-town status` — extended with one-line patch health summary
4. `gsd-town help` — updated to include doctor command
5. `test/cli/cli.test.js` — doctor tests: exits 0 or 1 (no crash), --fix exits 0

## Key Checks

- execute-phase.md contains gastown.sh source line (marker: `source` + `gastown.sh`)
- `~/.claude/skills/gsd-town-setup/` directory exists

## Implementation Notes

- Reuse postinstall.js constants (SKILL_DEST, EXECUTE_PHASE, gastown.sh marker) by requiring the file — but postinstall.js has side effects on require. Use a subprocess call instead: `node bin/postinstall.js`
- Doctor check logic is pure Node fs — no bash required
- Keep deviation from plan if postinstall.js needs minor refactor to be module-safe

# Phase 14 Plan 01: Gastown Drift Detection

## Objective

Catch "this command went away" or "this flag was removed" breakage in gastown upstream before it
silently breaks GSD-Town dispatch. Implement a fast, no-daemon API-surface smoke test in CI
plus a local bump workflow for safe submodule upgrades.

## Context

- `vendor/gastown` is pinned at `63bf531f285159edfe207dcedab1962d93a357b6`
- GSD-Town calls: `gt sling --no-convoy --no-merge`, `gt convoy stage --launch --json`, `gt polecat list --json`, `gt done`, `gt daemon status`
- CI runs on ubuntu-latest; no gastown daemon is available
- Existing CI job: `.github/workflows/test.yml` (npm test, unit/bash tests only)

## Tasks

### Task 1: Drift smoke test (test/drift/01-api-surface.sh)

Create `test/drift/01-api-surface.sh` that:
- Does NOT require a running daemon
- Verifies pinned `vendor/gastown` gt binary has expected CLI commands via `--help` only
- Checks: `gt sling --help`, `gt convoy stage --help`, `gt polecat list --help`, `gt done --help`, `gt daemon status --help`
- Checks help output for critical flags: `--json`, `--launch`, `--no-convoy`, `--no-merge`
- Optionally checks `bd --help` if built during the same build step
- Exits 0 on success, 1 if any drift detected
- Prints per-check results for easy diagnosis

### Task 2: CI integration (build-gastown-drift job)

Extend `.github/workflows/test.yml` with a parallel `build-gastown-drift` job:
- Checks out with `submodules: recursive`
- Sets up Go via `actions/setup-go@v5`
- Builds `vendor/gastown` via `make build` (skip daemon restart step of `make install`)
- Runs `bash test/drift/01-api-surface.sh` with GT binary path set
- Marked `continue-on-error: true` (advisory, not blocking)

### Task 3: README Gastown Compatibility section

Add `## Gastown Compatibility` section to README.md with:
- Known-good SHA (`63bf531f285159edfe207dcedab1962d93a357b6`)
- Submodule bump workflow (manual steps)
- Reference to `scripts/bump-gastown.sh`

### Task 4: Bump script (scripts/bump-gastown.sh)

Create `scripts/bump-gastown.sh`:
- Takes a commit SHA or branch as argument
- Updates the submodule to that ref
- Runs `test/drift/01-api-surface.sh` locally to verify
- Shows diff of `gt --help` output (old vs new) to surface behavior changes
- Does NOT auto-commit — leaves user to review and commit

## Verification

- `bash test/drift/01-api-surface.sh` exits 0 with current pinned submodule
- CI YAML is valid (use `act` or visual check)
- `bash scripts/bump-gastown.sh` prints usage when called with no args
